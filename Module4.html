MODULE 4 SECURE SOFTWARE DEVELOPMENT NOVEMBER 2022

In this module, we will explore the security risks associated with programming languages from the perspectives of design and architecture approaches, programming paradigms, testing, the impact and implications of operating systems and libraries on software development, the security risks associated with distributed systems and APIs, and future trends in the design of secure software and systems. Throughout these sections, the concept of the secure software development life cycle will be emphasized.

Unit-1: Introduction to Secure Software Development Unit 1 serve as the foundation for a number of key topics that will be utilized throughout the module. This begins with an evaluation of the management approaches for supporting software development, taking into account the shortcomings of more classic approaches such as waterfall and the strategies relevant to more contemporary approaches such as agile. The Unified Modelling Language (UML) will be examined in Week 1, with information gained being useful throughout the curriculum in respect to the modelling of software deployments from their design phase.

Learning Objectives:
1. Examine the waterfall and agile approaches to software development, focusing on the consequences of producing secure software using each method.
2. Become acquainted with the Unified Modelling Language and how it may be used to support software development.
3. Become acquainted with the Unified Modelling Language and how it may be used to support software development.
4. Acquire an understanding of the industry standards required to design secure software.
5. Recognize the significance of fostering a culture of risk awareness inside an organization.

Learning Outcome:
 1. Identify relevant scholarly material which explores the manner in which secure software may be produced utilizing an agile process.
2. Decompose an activity into a series of tasks which can be represented using a flow chart from the UML portfolio of modelling approaches.
3. Describe the significance of fostering a risk-aware culture inside an organization in addition to the development of secure software.

Unit 2: Modeling in UML to Support Secure System Planning

We will get some practical skills in the design of UML flowcharts during the second week. Flowcharts are utilized during the design phase of the SDLC and can be used thereafter to facilitate communication between stakeholders and document the final product.

Learning Objectives:
1. Gain an understanding of the prospects for incorporating software security development techniques at each level of the agile SDLC.
2. Recognize the variety of software security issues that can arise.

Learning Outcome:
 1. Review the scholarly literature (e.g., published by IEEE, ACM, Elsevier, or ScienceDirect) on agile software development that prioritizes security.
2. Become acquainted with the terminology and concepts of an ISO/IEC-published security standard.

Unit 3: Introduction to Programming Languages We will spend some time this week researching programming languages, examining the history, concepts, and design that lead to the languages we use today. Programming languages are a key notion in computer science; they are the primary instrument used by computer scientists and software developers to design, define, construct, and implement the tools we use today.

Learning Knowledge:
1. Review the history and fundamental ideas of programming languages.
2. Examine language ideas using Python as an illustration.
3. Describe the best practices and strategies for resolving typical security problems.

Learning Outcome:
 1. Describe many significant milestones in the evolution of programming languages.
2. Describe a few of the paradigms that define the many sorts of languages.
3. Describe the fundamental principles that govern the operation of programming languages.
4. Discuss significant programming issues and suggested best practices.
